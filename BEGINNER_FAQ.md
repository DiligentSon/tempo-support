# Tempo Beginner FAQ (Non-Developer Friendly)

This document is meant for beginners who aspire to contribute to the Tempo ecosystem
but have yet to gain deep developer experience.

## Who can use this?
- New contributors
- Community members who want to help
- Non-Rust developers
- Documentation contributors

## Is Rust needed to be written to contribute?
No.
You contribute by:
- Assiting beginners to understand Tempo concepts
- Improving documentation
- Writing FAQs
- Reporting unclear setup steps

## What do beginner-friendly ways to contribute look like?
- Adding explanations to README files
- Improving onboarding documentation
- Creating step-by-step guides
- Asking good questions in Issues
- Suggesting UX or documentation improvements

## Is running a node a prerequisite to known as a contributor?
No.
Most early contributions do not require running a node or spending money.

## How do contributions get recognized?
Tempo maintainers review all merged pull requests.
Only valid contributions help improve the ecosystem and may be considered for future incentives.

## Where is advisable for beginners to start?
- tempo-support repository
- GitHub Issues
- Documentation improvements

## I am new. Is my contribution welcome?
Yes.
Tempo is open-source and contributions of all sizes are welcome.

Small improvements matter.

This FAQ was created by a beginner contributor to assist others start with confidence.
